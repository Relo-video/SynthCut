# SynthCut MCP Client Setup

This file explains how users connect SynthCut by Relo to AI clients that support
MCP.

---

## Developer Setup (running from source)

If you are running SynthCut from source with `npm run dev` or `npm start`, use
the built MCP package directly with plain `node` — no Electron binary needed.

**Prerequisites:** build the packages first:

```cmd
npm run build
```

This produces `packages/mcp/dist/index.js`.

### Claude Code (dev)

```cmd
claude mcp add -s user ai-video-editor -- node "REPO_ROOT\packages\mcp\dist\index.js"
```

Replace `REPO_ROOT` with the absolute path to your clone, e.g.:

```cmd
claude mcp add -s user ai-video-editor -- node "C:\Users\Dhruv\Desktop\AI_native_editor\packages\mcp\dist\index.js"
```

### Claude Desktop / other clients (dev)

```json
{
  "mcpServers": {
    "ai-video-editor": {
      "command": "node",
      "args": ["C:\\Users\\Dhruv\\Desktop\\AI_native_editor\\packages\\mcp\\dist\\index.js"]
    }
  }
}
```

> **Note:** The dev core must already be running (`npm run core` or `npm start`)
> before the MCP server can connect, or start the Electron app first so it boots
> the core automatically.

> **Path changes:** Dev and installed-app MCP paths are different. If you switch
> between dev and the installed app, remove and re-add the MCP entry with the
> correct path.

## Installed App Setup

For users who installed SynthCut from the `.exe` installer or portable `.zip`,
the safest setup is:

1. Open SynthCut.
2. Click **Connect AI**.
3. Copy the generated MCP config.
4. Paste it into the AI client's MCP config file.
5. Fully restart the AI client.

Do not tell installed users to run `node packages/mcp/dist/index.js`. That path
is only for developers running SynthCut from source. Installed users should use
the config generated by SynthCut because it points to the bundled app runtime.

## Claude Desktop

Windows:

```text
%APPDATA%\Claude\claude_desktop_config.json
```

macOS:

```text
~/Library/Application Support/Claude/claude_desktop_config.json
```

Paste the generated `mcpServers` JSON from SynthCut:

```json
{
  "mcpServers": {
    "ai-video-editor": {
      "command": "PATH_FROM_SYNTHCUT_CONNECT_AI",
      "args": ["MCP_INDEX_PATH_FROM_SYNTHCUT_CONNECT_AI"],
      "env": {
        "ELECTRON_RUN_AS_NODE": "1"
      }
    }
  }
}
```

Then fully quit and restart Claude Desktop.

## Claude Code

Recommended command:

```powershell
claude.cmd mcp add -s user ai-video-editor -e ELECTRON_RUN_AS_NODE=1 -- "PATH_FROM_SYNTHCUT_CONNECT_AI" "MCP_INDEX_PATH_FROM_SYNTHCUT_CONNECT_AI"
```

Project-scoped config can also live in:

```text
<project>/.mcp.json
```

That file uses the same `mcpServers` JSON shape as Claude Desktop.

## Codex

Global config file:

Windows:

```text
C:\Users\<user>\.codex\config.toml
```

macOS/Linux:

```text
~/.codex/config.toml
```

Codex uses TOML, not JSON:

```toml
[mcp_servers.ai-video-editor]
command = "PATH_FROM_SYNTHCUT_CONNECT_AI"
args = ["MCP_INDEX_PATH_FROM_SYNTHCUT_CONNECT_AI"]

[mcp_servers.ai-video-editor.env]
ELECTRON_RUN_AS_NODE = "1"
```

```powershell
codex.cmd mcp add ai-video-editor --env ELECTRON_RUN_AS_NODE=1 -- "PATH_FROM_SYNTHCUT_CONNECT_AI" "MCP_INDEX_PATH_FROM_SYNTHCUT_CONNECT_AI"
```

## What The AI Client Sees

When an AI client connects to SynthCut's MCP server, the server exposes its tools
through standard MCP tool discovery. The client receives each tool's:

- name, such as `import_video`, `append_clip`, `trim_clip`, `render_preview`,
  `export_video`, `generate_captions`, or `timeline_summary`
- description, explaining what the tool does
- input schema, describing the required and optional parameters
- result content, usually text JSON, and sometimes images for frame/thumbnail
  tools

SynthCut's MCP server registers the editor's core RPC methods as MCP tools. That
means the AI and the desktop UI operate the same editing engine and the same
project state.

## How Tool Inputs Work

If a tool needs input, the AI model should call that tool with JSON that matches
the tool's input schema.

Example:

```json
{
  "path": "C:\\Users\\User\\Videos\\clip.mp4"
}
```

For frame-based edit operations, positions and durations are integer frame
counts, not seconds. The AI can use tools such as `get_state` or
`timeline_summary` first to inspect the project, find clip IDs, track indexes,
asset IDs, frame rate, and timeline structure. It can then pass those IDs and
frame numbers into editing tools.

If the AI sends invalid input, the schema validation fails or the tool returns a
tool error. The model should read the error, correct the JSON, and call the tool
again.

Long-running tools such as preview rendering, export, captions, auto-reframe,
and stabilization can report progress to MCP clients that support progress
notifications.
